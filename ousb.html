OUSB Process Control System

Low-Level C++ Hardware Interface

📋 The Engineering Challenge

The goal of this project was to develop a command-line interface (CLI) in C++ that interacts with the OUSB (Open Universal Serial Board) hardware simulator. The system reads 8-bit digital inputs and 10-bit analog inputs to update hardware registers.

🛠 Technical Implementation

1. Robust Error Precedence Hierarchy

I implemented a strict error-handling hierarchy to ensure system stability. If multiple errors occur, the system reports only the highest-priority error:

CodePriorityDescription
PHighestParameter Error: Incorrect number of arguments.
XHighNumerical Error: Input is not a valid number.
RMediumRange Error: Value is outside 0–255 limit.
VMediumValidation Error: Logic failure (Switch < Pot).
HLowHardware Error: Device failed to respond.

2. Intelligent Data Validation

A key feature is the expocnentialcheck function. It manually parses strings to validate numerical digits, scientific notation (E/e), and polarity signs (+/-) before conversion.

3. Bitwise Register Manipulation

The project involved manipulating 8-bit binary strings to represent hardware states. I wrote logic to convert integers into Standard and Reverse Binary formats to interact directly with hardware LED registers.

🧠 Key Skills Demonstrated

  • Systems Programming: Managing data types and memory for hardware interaction.
  • Industrial Logic: Implementing error precedence for reliability.
  • Register Control: Direct manipulation of PINC and ADC values.
