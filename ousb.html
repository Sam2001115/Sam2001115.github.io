PROJECT: OUSB_SIMULATOR

01. Objective

To develop a robust C++ command-line simulator for the Open USB (OUSB) board, ensuring perfect synchronization between decimal inputs and binary register outputs within a strict 24-hour development window.

02. Technology Used

C++ Standard Library Register-Level Logic File I/O Handling Data Validation Algorithms Command Line Interface (CLI)

03. The Challenge

The main difficulty lay in maintaining **Data Integrity** across different numeric bases. The system had to validate that a decimal string provided by a file matched a binary string exactly, while also ensuring those values didn't exceed the physical bit-limits of the 8-bit (0-255) and 10-bit (0-1023) hardware registers. Any single failure in the logic chain—from file reading to character parsing—required a specific, immediate "Z" error code output.

04. Resolution

I architected a Priority-Based Validation Pipeline. Instead of nested if-statements, the code runs through sequential checks (File Existence → Numeric Validation → Bit Length → Binary-to-Decimal Sync). If any layer fails, the program terminates safely with the "Z" code to protect potential hardware connections.

// Critical Synchronization Check if (decimalValue != binaryDecimalValue) { cout << "Z" << endl; return 0; } // Register Overflow Protection if (bitLength == _8bit && (decimalValue < 0 || decimalValue > 255)) { cout << "Z" << endl; return 0; }

By implementing custom string-to-integer and binary-to-decimal conversion functions rather than relying on high-level libraries, I ensured the simulator remained lightweight and compatible with low-level embedded environments.
