PROJECT: ROOM_MONITOR_DEVICE

01. Objective

Monitoring my room for any intrusion while I'm out. The device detects movement and sends real-time alerts to my phone via Telegram, including the duration of the intrusion.

02. Technology Used

ESP32 DevKit V1 HC-SR04 Ultrasonic Sensor 1.3" I2C OLED Display Telegram Bot API Healthchecks.io (Pinging) Push Buttons (Input)

03. Challenges

1. Connectivity Awareness: If the device is turned off or loses power, there was no way to know, leaving the room unprotected without notice.

2. Dynamic Placement: The detection range needed to be adjusted constantly in the code because the device doesn't have a fixed place, making it inefficient to re-flash the firmware every time I moved it.

04. Resolution

I utilized Healthchecks to send an automated email if the device goes offline by tracking the last ping sent by the ESP32.

To solve the range issue, I added two push buttons that allow for manual adjustment of the ultrasonic threshold directly on the device. The OLED display provides real-time feedback on the current range settings and connection status.

When movement is detected within the set range, the ESP32 communicates with a Telegram Bot on my phone to provide instant security updates and timestamps.

05. Results

✅ The system has been running for 3 months with 99.9% uptime, alerting me within 5 seconds of any intrusion. Healthchecks.io now sends a downtime alert if the device misses three consecutive pings, ensuring I never unknowingly leave the room unprotected.
